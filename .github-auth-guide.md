# GitHub Authentication Setup Guide

## Current Issue
```
remote: Invalid username or token
fatal: Password authentication is not supported for Git operations
```

GitHub no longer accepts password authentication. You need to use either:
1. Personal Access Token (PAT) - Recommended for quick setup
2. SSH Keys - Recommended for long-term use

---

## Option 1: Personal Access Token (PAT) 🔑

### Step 1: Create a PAT on GitHub

1. Go to: https://github.com/settings/tokens
2. Click **"Generate new token"** → **"Generate new token (classic)"**
3. Name it: `feedback-forge-dev`
4. Set expiration: **90 days** (or custom)
5. Select scopes:
   - ✅ `repo` (Full control of private repositories)
   - ✅ `workflow` (Update GitHub Action workflows)
6. Click **"Generate token"**
7. **COPY THE TOKEN** (you won't see it again!)

### Step 2: Configure Git to Use the PAT

```bash
# Store credentials (saves PAT securely)
git config --global credential.helper store

# Now push - it will ask for credentials ONCE
git push

# When prompted:
# Username: zodraz
# Password: <paste your PAT here>
```

After this, Git will remember your PAT.

---

## Option 2: SSH Keys (Better for Long-term) 🔐

### Step 1: Check if you have SSH keys

```bash
ls -la ~/.ssh/
# Look for: id_rsa.pub, id_ed25519.pub, or similar
```

### Step 2: Generate new SSH key (if needed)

```bash
ssh-keygen -t ed25519 -C "fake@fake.com"
# Press Enter for default location
# Enter passphrase (optional but recommended)
```

### Step 3: Add SSH key to ssh-agent

```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

### Step 4: Copy SSH key to clipboard

```bash
cat ~/.ssh/id_ed25519.pub
# Copy the entire output
```

### Step 5: Add to GitHub

1. Go to: https://github.com/settings/keys
2. Click **"New SSH key"**
3. Title: `feedback-forge-wsl`
4. Paste your key
5. Click **"Add SSH key"**

### Step 6: Change remote to SSH

```bash
# Switch from HTTPS to SSH
git remote set-url origin git@github.com:zodraz/feedback-forge.git

# Test connection
ssh -T git@github.com

# Now push works!
git push
```

## Quick Fix (Right Now)

If you need to push immediately:

```bash
# Use credential helper
git config --global credential.helper store

# Push (will prompt for PAT)
git push -u origin main

# Enter:
# Username: zodraz
# Password: <your PAT from GitHub>
```

## Verify Setup

```bash
# Check remote URL
git remote -v

# If using HTTPS: https://github.com/zodraz/feedback-forge.git
# If using SSH:   git@github.com:zodraz/feedback-forge.git

# Test authentication
git ls-remote
```

---

## Security Notes

- ✅ PATs are safer than passwords
- ✅ SSH keys are more secure than PATs
- ✅ Use `credential.helper store` to save credentials
- ❌ Never share your PAT or private SSH key
- ⚠️ Set PAT expiration dates
- 🔄 Rotate PATs regularly

---

**Choose Option 1 (PAT) for quick setup, or Option 2 (SSH) for better long-term security.**
